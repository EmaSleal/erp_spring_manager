OBJETIVO DEL PROYECTO
================================================================================

Transformar WhatsApp Orders Manager en un ERP profesional con:

1. ✅ Dashboard principal con módulos tipo Odoo
2. ✅ Barra de navegación superior con menú de usuario
3. ✅ Sistema de perfil de usuario
4. ✅ Navegación intuitiva entre módulos
5. ✅ Diseño responsive y moderno
6. ✅ Integración con módulos existentes (Clientes, Productos, Facturas)

RIESGOS Y CONSIDERACIONES
================================================================================

⚠️ RIESGOS IDENTIFICADOS:
──────────────────────────
1. Compatibilidad con funcionalidad existente
   Mitigación: Testing exhaustivo después de integración

2. Curva de aprendizaje de Bootstrap/Font Awesome
   Mitigación: Documentación clara, ejemplos en MOCKUPS_VISUALES.txt

3. Tiempo de desarrollo puede extenderse
   Mitigación: Priorización clara, Sprint 1 bien definido

4. Responsive design en todos los dispositivos
   Mitigación: Testing en múltiples breakpoints

5. Performance con muchas estadísticas
   Mitigación: Caché, optimización de consultas

📋 DEPENDENCIAS:
────────────────
- Bootstrap 5 (CDN)
- Font Awesome 6 (CDN)
- SweetAlert2 (CDN)
- Spring Security (ya instalado)
- Thymeleaf (ya instalado)

No requiere instalación de nuevas dependencias en pom.xml para Sprint 1.

================================================================================
CRITERIOS DE ACEPTACIÓN (SPRINT 1)
================================================================================

El Sprint 1 se considerará COMPLETADO cuando:

✅ 1. Usuario puede hacer login y es redirigido automáticamente a /dashboard
✅ 2. Dashboard muestra:
     - 4 widgets con estadísticas correctas (Clientes, Productos, Facturas, Pagos)
     - Grid de módulos con al menos 10 tarjetas
     - Módulos activos son clickeables y redirigen correctamente
     - Módulos inactivos muestran alerta "Próximamente"

✅ 3. Navbar está presente en TODAS las páginas y muestra:
     - Logo y nombre de la aplicación
     - Botón "Dashboard" (si no está en dashboard)
     - Nombre de usuario con iniciales
     - Rol del usuario (badge)
     - Dropdown funcional con todas las opciones

✅ 4. Dropdown de usuario incluye:
     - "Ver mi perfil" → funcional
     - "Configuración" → visible solo para ADMIN
     - "Mi actividad" → link (puede ser placeholder)
     - "Cerrar sesión" → funcional con confirmación

✅ 5. Perfil de usuario:
     - Ver perfil muestra toda la información del usuario
     - Editar perfil permite modificar: nombre, email, teléfono
     - Cambiar contraseña funciona correctamente
     - Validaciones client-side y server-side

✅ 6. Logout:
     - Muestra confirmación con SweetAlert2
     - Cierra sesión correctamente
     - Invalida sesión y cookies
     - Redirige a /auth/login con mensaje de éxito

✅ 7. Módulos existentes (Clientes, Productos, Facturas):
     - Integrados con nuevo navbar
     - Mantienen su funcionalidad original
     - Tienen botón "Volver a Dashboard"

✅ 8. Responsive:
     - Dashboard funcional en móvil (< 576px)
     - Navbar colapsable en móvil
     - Grid de módulos adapta columnas según dispositivo
     - Widgets de estadísticas se apilan verticalmente en móvil

✅ 9. No hay errores:
     - Consola del navegador limpia (sin errores JS)
     - Sin errores 404 (recursos no encontrados)
     - Sin errores 500 (server errors)
     - Sin warnings de Thymeleaf

✅ 10. Seguridad:
      - Rutas protegidas correctamente
      - Solo ADMIN ve módulos de administración
      - CSRF tokens en todos los formularios
      - Logout seguro
